# Local Media Viewer

Локальное веб-приложение для просмотра фото и видео из директории на компьютере или сервере в локальной сети.

## Возможности

- вход по пинкоду из `.env`;
- авторизация хранится на время браузерной сессии, а frontend-флаг хранится в `sessionStorage`;
- дерево папок внутри `MEDIA_ROOT`;
- сетка фото и видео с маленькими, средними и большими превью;
- загрузка файлов и папок через зону загрузки в дереве или drag & drop;
- создание новых папок из дерева папок;
- перемещение файлов между папками и перемещение папок внутри дерева;
- удаление файлов и папок после подтверждения;
- полноэкранный viewer с листанием мышью и клавиатурой;
- zoom, reset и pan для фото;
- видео с нативными controls и HTTP Range-запросами для перемотки;
- защита от path traversal и выдачи файлов вне `MEDIA_ROOT`.

## Установка

```bash
npm install
cp .env.example .env
```

Откройте `.env` и задайте свои значения:

```bash
MEDIA_ROOT=/path/to/media
PIN_CODE=1234
PORT=3000
```

`PIN_CODE` должен состоять только из цифр, длина от 4 до 16 символов.

## Dev-режим

```bash
npm run dev
```

В dev-режиме backend запускается на `http://localhost:3000`, Vite-клиент на `http://localhost:5173`.
Открывайте в браузере:

```text
http://localhost:5173
```

## Production

Собрать клиент:

```bash
npm run build
```

Запустить локальный сервер:

```bash
npm start
```

После запуска откройте:

```text
http://localhost:3000
```

Если в `.env` указан другой `PORT`, используйте его.

## Поддерживаемые файлы

Фото: `jpg`, `jpeg`, `png`, `gif`, `webp`, `avif`.

Видео: `mp4`, `webm`, `mov`, `m4v`.

Остальные файлы игнорируются.

## Управление файлами

### Загрузка файлов

Откройте нужную папку в дереве слева и используйте зону загрузки внизу панели папок.
Клик по зоне открывает выбор файлов. Можно выбрать один или несколько файлов. Поддерживаемые фото и видео будут сохранены в текущую выбранную папку, после загрузки список обновится.

### Загрузка папок

Откройте папку назначения и перетащите папку в зону загрузки внизу дерева.
Также в зоне загрузки есть действие выбора папки. При загрузке папки сохраняется её внутренняя структура: подпапки будут созданы внутри текущей выбранной папки.

Например, если в текущую папку загрузить `Trip/Day 1/photo.jpg`, файл появится как:

```text
Current folder/Trip/Day 1/photo.jpg
```

### Drag & Drop

Файлы и папки можно перетаскивать в зону загрузки снизу дерева.
При перетаскивании зона подсвечивается. Для папок структура подпапок сохраняется так же, как при выборе папки.

### Перемещение

### Создание папок

В дереве папок наведите на нужную папку и нажмите `+`.
Введите имя новой папки. Если папка с таким именем уже существует, приложение создаст уникальное имя, например `Folder (1)`.

### Перемещение

Файл из сетки справа можно перетащить на папку в дереве слева.
После перемещения файл исчезнет из текущей папки и появится в папке назначения.

Папки можно перетаскивать внутри дерева папок. Корневую папку перемещать нельзя. Папку нельзя переместить внутрь самой себя или в свою подпапку.

### Удаление

У карточек медиафайлов кнопка удаления появляется при наведении. Перед удалением показывается подтверждение.

У папок кнопка удаления появляется при наведении в дереве. Корневую папку удалить нельзя. При удалении папки удаляется всё её содержимое, поэтому перед операцией показывается отдельное подтверждение.

### Конфликты имён

Если файл или папка с таким именем уже существует, приложение не перезаписывает его.
Вместо этого создаётся безопасное уникальное имя:

```text
photo.jpg
photo (1).jpg
photo (2).jpg
Folder
Folder (1)
```

## API

- `POST /api/login` с JSON `{ "pin": "1234" }`
- `POST /api/logout`
- `GET /api/tree`
- `GET /api/media?path=/relative/path`
- `POST /api/upload?path=/relative/path`
- `POST /api/folder` с JSON `{ "parentPath": "/target", "name": "New folder" }`
- `POST /api/move-file` с JSON `{ "from": "/old/file.jpg", "toDir": "/target" }`
- `POST /api/move-folder` с JSON `{ "from": "/old/folder", "toDir": "/target" }`
- `DELETE /api/media` с JSON `{ "path": "/file.jpg" }`
- `DELETE /api/folder` с JSON `{ "path": "/folder" }`
- `GET /media/...`

Все API-запросы, кроме login/logout, и все файлы `/media/...` требуют авторизации.

# Cache Architecture

Кеширование в приложении разделено на три части:

- превью медиафайлов в `CACHE_DIR/previews`;
- легковесный индекс метаданных в `CACHE_DIR/metadata.json`;
- очередь генерации превью с ограничением параллелизма.

Оригинальные медиафайлы не кешируются. Будущий кеш оригиналов должен быть отдельным слоем и не должен смешиваться с индексом превью.

## Основные модули

- `server/cache/config.js` читает настройки кеша из переменных окружения.
- `server/cache/cacheKeys.js` строит ключи кеша с учетом версии, пути, размера, времени изменения, типа и формата превью.
- `server/cache/mediaMetadata.js` сохраняет базовые метаданные файлов для повторного открытия директорий.
- `server/cache/metadataStore.js` хранит индекс в локальном JSON-файле. Его можно заменить на SQLite без изменения публичного слоя кеша.
- `server/cache/previewCache.js` проверяет актуальность превью, переиспользует готовые файлы и ставит генерацию в очередь.
- `server/cache/previewQueue.js` ограничивает число параллельных задач и дедуплицирует одинаковые запросы.
- `server/cache/cacheCleanup.js` удаляет старые превью по LRU при превышении лимита размера.
- `server/cache/generators/*` содержит генераторы превью для изображений и видео.

## Запрос превью

1. Интерфейс лениво запрашивает `/preview/<path>?size=480`, когда карточка появляется рядом с viewport.
2. Сервер проверяет путь, тип файла и `stat` оригинала.
3. `previewCache` строит ключ по `CACHE_VERSION`, реальному пути, размеру файла, `mtimeMs`, типу, размеру, формату и генератору превью.
4. Если индекс и файл превью актуальны, готовый файл отдается сразу.
5. Если превью нет, задача ставится в очередь. Повторные запросы того же ключа ждут ту же задачу.
6. После генерации индекс получает `lastAccessedAt` и размер кеш-файла.
7. Очистка запускается в фоне и не блокирует ответ.

## Открытие директории

1. `listMedia` читает директорию и фильтрует поддерживаемые файлы.
2. Для каждого файла берется `stat`, чтобы проверить актуальность.
3. `mediaMetadata` возвращает актуальные базовые метаданные из индекса или записывает свежие.
4. Генерация превью при открытии директории не запускается.

## Инвалидация

Запись считается актуальной только при совпадении:

- версии кеша;
- реального пути;
- размера файла;
- `mtimeMs`;
- типа медиа;
- размера превью;
- формата превью.

При изменении файла старый кеш не используется. Устаревшие файлы могут быть удалены фоновой LRU-очисткой.
